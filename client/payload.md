# 요청 파라미터

## Payload  파라미터

|         파라미터        | 설명                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| :-----------------: | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
|   application\_id   | <p><code>String</code> / <strong><code>필수</code></strong></p><p>부트페이로부터 발급받은 Application ID</p><p>(<a href="../master.md#undefined">연동 전 설정하기</a>에서 확인 가능)</p>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
|         name        | <p><code>String</code> / <strong><code>필수</code></strong></p><p>결제 창에 노출될 판매할 상품명 이름 <br>(특수문자 입력시 결제가 안될 수 있음, 한글 기준 12글자 이내 추천)</p>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|        price        | <p><code>Integer</code> / <strong><code>필수</code></strong><br>결제할 금액을 숫자로 입력합니다. 최소 100원이상의 금액을 입력해야 합니다. <br>( 결제 수단에 따라 최소 금액이 3,000원인 경우도 있음 )</p>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|      order\_id      | <p><code>String</code> / <strong><code>필수</code></strong></p><p>개발사에서 관리하는 주문번호 고유 값(PK)</p><p>주문번호는 반드시 <strong>고유한 값 (Unique)</strong> 해야 합니다.<br>고유한 값이 아닌 경우에도 결제는 가능하지만 부트페이 서버에서는 <strong>결제 Transaction</strong>을 지정해주신 주문번호 고유 값으로 설정하여 처리하고 있기 때문에 중복 결제 및 부정적인 결제 개입의 막기 위해 반드시 고유값으로 설정할 것을 권장합니다</p>                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|      tax\_free      | <p><code>Integer</code><br>비과세 금액 설정시 해당 필드를 사용합니다. <br>비과세 품목에 해당하는 상품의 금액입니다. <br>예) price=5000, tax_free=2500 일 경우, 결제건 5천원 중 2,500원은 비과세 상품인 결제건을 의미함</p>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
|          pg         | <p><code>String</code></p><p><a href="https://admin.bootpay.co.kr/install/method">부트페이 관리자 - 결제수단</a>에서 활성화된 PG가 결제창에 노출됩니다.</p><ol><li>값을 지정할 경우 - <a href="pg/">일반결제 </a><br>ex) danal, kcp, inicis 등 <a href="pg/#pg">코드표 참조</a></li><li>값을 지정하지 않을 경우 - <a href="total.md">통합결제</a></li></ol>                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
|        method       | <p></p><p><code>String</code></p><p><a href="https://admin.bootpay.co.kr/install/method">부트페이 관리자 - 결제수단</a>에서 활성화된 결제수단이 결제창에 노출됩니다.</p><ol><li><p>값을 지정할 경우 - <a href="pg/">일반결제 </a></p><ul><li><code>card</code> - 신용카드 결제</li><li><code>phone</code> - 휴대폰 소액결제</li><li><code>bank</code> - 실시간 계좌이체</li><li><code>vbank</code> - 가상계좌</li><li><code>auth</code> - 본인인증 ( 현재는 다날만 가능 )</li><li><code>card_rebill</code> - 카드 정기결제</li><li><code>kakao</code> - (일반PG사) 카카오페이</li><li><code>npay</code> - (일반PG사) 네이버페이</li><li><code>easy</code> - (간편결제사) 카카오, 페이코등의 간편결제</li></ul><p></p></li><li><p>값을 지정하지 않을 경우 - <a href="total.md">통합결제</a></p><p><code>pg</code>, <code>method</code> 값을 모두 지정하지 않을 경우, 활성화된 PG, 결제수단이 통합결제에 모두 반영됩니다.</p></li></ol> |
| show\_agree\_window | <p><code>Integer</code></p><p>결제 전 정보 수집에 관련된 동의창을 띄울지 말지 선택합니다</p><ul><li><code>1</code> - 동의창을 띄운다.</li><li><code>0</code> - 동의창을 띄우지 않는다.</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
|        items        | <p><code>Array&#x3C;Object></code></p><p>판매할 아이템 정보 </p><p>통계 정보에 이용되므로 입력하지 않을 경우 카테고리 및 아이템 판매에 관련된 통계 데이터를 볼 수 없습니다.</p><ul><li><code>item_name</code> - 아이템 이름 (string)</li><li><code>qty</code> - 상품 판매된 수량 (integer)</li><li><code>unique</code> - 개발사에서 관리하는 상품의 고유 PK (string)</li><li><code>price</code> - 상품 하나당 판매 가격 (integer)</li><li><code>cat1</code> - 카테고리 상위 (string)</li><li><code>cat2</code> - 카테고리 중위 (string)</li><li><code>cat3</code> - 카테고리 하위 (string)</li></ul>                                                                                                                                                                                                                                                                       |
|      user\_info     | <p></p><p>구매자 정보 ( <code>payapp, kcp, inicis</code> 이용시 필수)</p><ul><li><code>username</code> - 구매자 명 (string)</li><li><code>email</code> - 구매자의 이메일 정보 (string)</li><li><code>addr</code> - 구매자의 주소 (string)</li><li><code>phone</code> - 구매자의 전화번호  (string)<br>* 페이앱은 필수 입력입니다.</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|        params       | <p><code>object</code></p><p>결제 후 똑같이 받을 파라메터들을 설정합니다. 결제가 완료된 후 어떤 결제인지 구분하는데는 <strong>order_id</strong>를 사용하지만, 그 결제에 대한 부가적인 정보를 params에 포함하여 보내면 좀 더 쉽게 처리할 수 있습니다. </p>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| account\_expire\_at | <p><code>String</code></p><p>가상계좌로 입금 가능한 마지막 날짜 (가상계좌 선택시에만 사용)</p><p><strong>'yyyy-mm-dd'</strong> 포맷으로 보내야 함 </p>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
|        extra        | <p>각 PG사의 옵션에 맞는 추가적인 파라메터 전달자<br><strong>아래 extra 옵션 추가적인 설명을 확인해주세요.</strong></p>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |



## Extra 파라미터

|           파라미터           | 설명                                                                                                                                                     |
| :----------------------: | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
|           popup          | <p><code>integer</code></p><p>결제창을 팝업으로 열 경우 1 로 지정 (0일 경우 iframe)</p><p>* 네이버페이의 경우 모바일 환경에서는 팝업은 필수입니다</p>                                           |
|          escrow          | <p><code>boolean</code> </p><p>에스크로 결제 여부를 보냅니다.</p>                                                                                                   |
|       expire\_month      | <p><code>integer</code></p><p>정기결제 제공 개월 수</p><p>( ex. 36 입력시 3년간 정기결제가 가능한 빌링키를 제공)</p><p>* 최대 기한은 카드에 명시된 만료일자까지 가능합니다.</p>                          |
|           quota          | <p><code>String</code> <br>카드 결제시 할부 개월수를 보여줍니다.</p><p>( ex. "0, 3, 4, 5, 6" - 일시불, 3개월, 4개월, 5개월, 6개월까지 보임 )</p><p>* 무이자 할부는 PG사 혹은 카드사 정책에 따릅니다.</p> |
| subscribe\_test\_payment | <p><code>boolean</code> </p><p>카드정기결제시 테스트 최소 금액 (100원, 페이앱은 1,000원) 결제를 하고 빌링키를 전달합니다.</p><p>* 카드 확인용으로 자동결제된 금액은 10분 뒤 자동 취소 됩니다.</p>                |
|    disp\_cash\_result    | <p><code>boolean</code><br>계좌이체, 가상계좌에서 현금영수증 처리 창을 보여줄 것인지 여부 <br>( 다날, KCP, 이니시스, 나이스페이만 가능 )</p>                                                    |
|       offer\_period      | <p><code>string</code></p><p>카드정기결제시 상품 결제 주기 설명 추가 기능 <br>( 페이앱, 나이스페이, 웰컴페이먼츠, 다날만 가능 )</p>                                                          |
|       seller\_name       | <p><code>String</code></p><p>PG 결제창의 판매자 이름을 설정합니다. ( 판매자명이 변경이 가능한 PG만 사용 가능, 일부 PG는 계정에 등록된 판매자명만 사용가능 )</p>                                         |
|           theme          | <p><code>String</code></p><p>통합 결제창 색상 지정 <br>(<code>purple, red, custom</code> 지정 가능)</p>                                                             |
|    custom\_background    | <p><code>String</code></p><p>theme가 custom인 경우 배경 색 지정 가능 <br>( ex: <code>#f2f2f2</code> )</p>                                                         |
|    custom\_font\_color   | <p><code>String</code></p><p>theme가 custom인 경우 폰트색 지정 가능 <br>( ex: <code>#333333</code> )</p>                                                          |
|         raw\_data        | <p><code>boolean</code></p><p><a href="../advance/pg.md">PG 오류 데이터 요청하기</a> 참조 </p><p>결제요청 실패시 PG사의 response data를 그대로 받길 원할때 true </p>                |
